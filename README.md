# 在线 PDF 阅读器方案（设计稿）

本仓库提供在线 PDF 阅读器的系统设计、接口草案与前端交互规划，满足以下需求：
- 兼容桌面与移动端的响应式阅读体验；
- 管理员（超级/注册）可上传、分享 PDF；
- 分享支持 **试读模式**（前 N 页公开）与 **浏览模式**（收件人邮箱认证）；
- 浏览端全屏水印，严格禁止下载、打印、复制。

## 文件总览
- `docs/ARCHITECTURE.md`：总体架构、流程、安全与运维设计。
- `docs/API_SPEC.md`：REST API 草案，覆盖认证、上传、分享、阅读等接口。
- `docs/FRONTEND.md`：前端技术栈、页面/组件规划与交互防护策略。

## 快速阅读
1. 先阅读 `ARCHITECTURE.md` 了解角色、流程与安全方案；
2. 根据 `API_SPEC.md` 实现后端服务、数据库与对象存储；
3. 参考 `FRONTEND.md` 构建管理员后台与访客阅读器，落实水印、防打印/下载策略；
4. MVP 完成后迭代可观测性、PWA、企业化能力。

## 安全与体验要点
- 服务端拆分：Auth、Document、Share、Reader、Audit、Notification；
- PDF 以分片/图像流式返回，结合短时签名 URL 与设备绑定 Token；
- Canvas + CSS 双水印，打印/下载/复制拦截并审计；
- 邮件魔法链接 + 验证码认证，支持访问次数、IP/设备限制；
- 响应式 UI，移动端提供手势翻页与底部工具栏。
